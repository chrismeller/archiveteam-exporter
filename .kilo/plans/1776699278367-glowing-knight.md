# Plan: Add Environment Variable Support for Cache TTLs

## Context
The separate cache durations (`ProjectsCacheDurationMinutes` / `StatsCacheDurationMinutes`) are already implemented. Now we need to make them configurable via environment variables, following the same pattern used for `USERNAME` and `PROJECTS_WHITELIST`.

## Change — `Program.cs` only

Read two new env vars and add them to the in-memory config dictionary (before the existing `Configure<ArchiveTeamOptions>` call):

```csharp
var projectCacheTtl = Environment.GetEnvironmentVariable("PROJECT_CACHE_TTL");
var statsCacheTtl = Environment.GetEnvironmentVariable("STATS_CACHE_TTL");

if (!string.IsNullOrEmpty(projectCacheTtl))
{
    inMemoryConfig[$"{ArchiveTeamOptions.SectionName}:ProjectsCacheDurationMinutes"] = projectCacheTtl;
}

if (!string.IsNullOrEmpty(statsCacheTtl))
{
    inMemoryConfig[$"{ArchiveTeamOptions.SectionName}:StatsCacheDurationMinutes"] = statsCacheTtl;
}
```

This follows the exact same pattern already used for `USERNAME` and `PROJECTS_WHITELIST`. The env vars contain integer minutes (matching the options properties), and the configuration binder handles the `string → int` conversion.
