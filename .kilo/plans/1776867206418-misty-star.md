# Plan: Remove Project List Fetch and Rename Whitelist to Projects

## Goal
1. Remove the HTTP call that fetches the full project list from `https://warriorhq.archiveteam.org/projects.json`.
2. Rename the `ProjectsWhitelist` configuration option to `Projects`.
3. Make `Projects` a required option that contains the comma-separated list of projects to include. The project list will be built directly from this configuration instead of being fetched and filtered.

## Files to Change

### 1. `src/ArchiveTeam.Exporter.ApiService/Options/ArchiveTeamOptions.cs`
- Rename property `ProjectsWhitelist` → `Projects`.
- Update the `ValidCommaSeparatedListAttribute` error message to reference `Projects` instead of `ProjectsWhitelist`.
- Make the validator reject null/empty strings so the option is effectively required (the `[Required]` attribute on the property handles null, but the custom validator should also reject whitespace-only strings).

### 2. `src/ArchiveTeam.Exporter.ApiService/Program.cs`
- Rename env var lookup from `PROJECTS_WHITELIST` to `PROJECTS`.
- Rename in-memory config key from `ProjectsWhitelist` to `Projects`.

### 3. `src/ArchiveTeam.Exporter.ApiService/appsettings.json`
- Rename `"ProjectsWhitelist"` → `"Projects"`.

### 4. `src/ArchiveTeam.Exporter.ApiService/appsettings.Development.json`
- Rename `"ProjectsWhitelist"` → `"Projects"`.

### 5. `src/ArchiveTeam.Exporter.ApiService/Services/ProjectService.cs`
- Remove `FetchProjectsAsync` entirely. Caching a static configuration string is pointless.
- In `GetProjectGaugesAsync`:
  - Build the `ArchiveTeamProject[]` directly from `_options.Projects` by splitting on commas, trimming, and mapping each entry to `new ArchiveTeamProject { Name = entry, Title = entry }`.
- Remove the project-related cache key constant (`CacheKey = "projects"`), since the project list no longer needs caching.
- Remove the private `FilterByWhitelist` method entirely.
- Remove the `ArchiveTeamProjectsResponse` dependency (using statement if any).
- Remove project-list-related cache refresh gauge updates (the `source="projects"` cache refresh metric is no longer meaningful since there is no fetch step).

### 6. `src/ArchiveTeam.Exporter.ApiService/Services/IProjectService.cs`
- Remove `FetchProjectsAsync` from the interface. Only `GetProjectGaugesAsync` remains.

### 7. `src/ArchiveTeam.Exporter.ApiService/Models/ArchiveTeamProjectsResponse.cs`
- Delete this file. It is no longer used once the project list fetch is removed.

## Unchanged Files
- `ArchiveTeamProject.cs` — model stays the same; we just populate it differently.
- `ProjectStatsResponse.cs` — unchanged.
- Tests and AppHost — no references to whitelist or project list fetch.

## Verification
- Build the solution (`dotnet build`).
- Ensure no compilation errors.
- Confirm that `FetchProjectsAsync` no longer calls an external URL to get the project list.
- Confirm the environment variable is now `PROJECTS` and the configuration key is `ArchiveTeam:Projects`.
