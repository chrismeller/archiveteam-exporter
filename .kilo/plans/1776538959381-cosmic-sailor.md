# Plan: Install hk via mise and configure qlty as a pre-commit hook

## Overview

Install the `hk` git hook manager using `mise`, then configure it to run `qlty fmt` (auto-format) as a pre-commit hook. This leverages the existing `.qlty/qlty.toml` configuration that's already set up in the repo.

## Steps

### 1. Install hk with mise

```sh
mise use hk
```

This adds `hk` to the project's `mise.toml` (which will be created if it doesn't exist) and installs it.

### 2. Create `hk.pkl` config file

Create `hk.pkl` in the project root with a pre-commit hook that runs `qlty fmt` on staged files:

```pkl
amends "package://github.com/jdx/hk/releases/download/v1.43.0/hk@1.43.0#/Config.pkl"

hooks {
    ["pre-commit"] {
        fix = true
        stash = "git"
        steps {
            ["qlty-fmt"] {
                glob = new { "*" }
                check = "qlty fmt --check {{files}}"
                fix = "qlty fmt {{files}}"
            }
        }
    }
}
```

**Note:** The version in the `amends` URL may need to be adjusted to match the installed hk version. We'll verify the installed version and use the correct URL.

### 3. Install the git hooks

```sh
hk install
```

This wires hk into `.git/hooks/pre-commit`.

### 4. Verify setup

```sh
hk check   # dry-run to verify the hook config works
```

### 5. Add `hk.local.pkl` to `.gitignore`

Append `hk.local.pkl` to `.gitignore` so local overrides aren't committed.

## Files Modified

| File | Action |
|---|---|
| `mise.toml` | Created (by `mise use hk`) |
| `hk.pkl` | Created (hk pre-commit hook config) |
| `.git/hooks/pre-commit` | Created (by `hk install`) |
| `.gitignore` | Updated (add `hk.local.pkl`) |

## Notes

- `hk` uses Apple's Pkl configuration language. The `amends` directive imports the hk schema so the config is type-checked.
- The `stash = "git"` setting ensures unstaged changes are stashed before running the hook and restored after.
- `fix = true` means the pre-commit hook will run the `fix` command (auto-format), not just `check`.
- `qlty fmt` will auto-format staged files before they're committed, using the linters/plugins configured in `.qlty/qlty.toml`.
- If a version mismatch occurs with the `amends` URL, `hk init` can be run to generate a template `hk.pkl` with the correct version.
