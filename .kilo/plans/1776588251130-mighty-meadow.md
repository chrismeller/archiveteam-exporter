# Plan: Configure Aspire to Generate Docker Compose via `aspire publish`

## Goal
Add the `Aspire.Hosting.Docker` publishing environment so that `aspire publish` generates Docker Compose artifacts for the project. Aspire will automatically containerize the .NET projects — no custom Dockerfile needed.

## Changes

### 1. Add `Aspire.Hosting.Docker` NuGet package to the AppHost
**File:** `src/ArchiveTeam.Exporter.AppHost/ArchiveTeam.Exporter.AppHost.csproj`

Add a `<PackageReference Include="Aspire.Hosting.Docker" Version="..." />` to the existing `<ItemGroup>`. The version should match the Aspire 13.2 family (e.g., `13.2.2`).

### 2. Register the Docker Compose environment in `AppHost.cs`
**File:** `src/ArchiveTeam.Exporter.AppHost/AppHost.cs`

Add `builder.AddDockerComposeEnvironment("compose");` before building the app. The `AddProject` call stays as-is — Aspire will automatically containerize it when publishing to Docker Compose.

```csharp
var builder = DistributedApplication.CreateBuilder(args);

builder.AddDockerComposeEnvironment("compose");

var apiService = builder.AddProject<Projects.ArchiveTeam_Exporter_ApiService>("apiservice")
    .WithExternalHttpEndpoints()
    .WithHttpHealthCheck("/health");

builder.Build().Run();
```

## Usage
After these changes, running `aspire publish -o docker-compose-artifacts` will generate Docker Compose files (including automatically building container images for the ApiService project).

## Notes
- No custom Dockerfile is required — Aspire handles containerization of .NET projects automatically.
- The `AddProject` call remains unchanged; the Docker Compose environment is additive.
- `Aspire.Hosting.Docker` is the official Aspire extension for Docker Compose publishing (introduced in Aspire 9.x).
