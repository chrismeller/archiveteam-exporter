# Plan: Add Cache Last-Refresh Metric and Update HELP Text

## Background

The exporter caches API responses from ArchiveTeam APIs with configurable TTLs:
- **Projects list** (`warriorhq.archiveteam.org/projects.json`): 30 min default
- **Project stats** (`v1.api.tracker.archiveteam.org/{project}/stats.json`): 1 min default

Currently, there is no metric exposing when cached data was last refreshed, and the metric HELP descriptions don't mention caching.

## Changes

### 1. Add `archiveteam_cache_last_refresh_timestamp_seconds` Gauge — `ProjectService.cs`

Add a new gauge to track the Unix timestamp of the last successful cache refresh:

```csharp
_archiveteamCacheLastRefreshGauge = Metrics.CreateGauge(
    "archiveteam_cache_last_refresh_timestamp_seconds",
    "Unix timestamp of the last successful data refresh from the ArchiveTeam API. Results are cached; this metric indicates when the cached data was last updated.",
    new GaugeConfiguration
    {
        LabelNames = ["source"]
    });
```

The `source` label will distinguish between the two cache types:
- `source="projects"` — the project list cache
- `source="stats"` — the per-project stats cache (set each time any project's stats are refreshed)

This gauge will be set to `DateTimeOffset.UtcNow.ToUnixTimeSeconds()` immediately after a successful API response is cached, in both `FetchProjectsAsync` and `FetchProjectStatsAsync`.

### 3. Files Modified

Only one file requires changes:

- **`src/ArchiveTeam.Exporter.ApiService/Services/ProjectService.cs`**
  - Add new `_cacheLastRefreshGauge` field and initialization in the constructor
  - Set the gauge after caching projects in `FetchProjectsAsync` (with label `source="projects"`)
  - Set the gauge after caching stats in `FetchProjectStatsAsync` (with label `source="stats"`)
  - Update all 7 existing gauge description strings

### 4. Example Output

After implementation, the `/metrics` endpoint would include:

```
# HELP archiveteam_cache_last_refresh_timestamp_seconds Unix timestamp of the last successful data refresh from the ArchiveTeam API. Results are cached; this metric indicates when the cached data was last updated.
# TYPE archiveteam_cache_last_refresh_timestamp_seconds gauge
archiveteam_cache_last_refresh_timestamp_seconds{source="projects"} 1.7451666e+09
archiveteam_cache_last_refresh_timestamp_seconds{source="stats"} 1.745166738e+09

# HELP archiveteam_projects_info Information about ArchiveTeam projects. Data is cached; see archiveteam_cache_last_refresh_timestamp_seconds for cache age.
...
```

This allows users to:
- Know data is cached by reading HELP text
- Alert on stale data by comparing `time() - archiveteam_cache_last_refresh_timestamp_seconds` against the configured TTL
- Distinguish freshness of project list vs. stats data independently
