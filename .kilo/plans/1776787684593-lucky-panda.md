# Plan: Parse domain_bytes.data and Expose as Metric

## Summary

The ArchiveTeam tracker API returns `domain_bytes` as a nested object with a `data` property (e.g., `{"domain_bytes": {"data": 3.79e+15}}`), not as a flat dictionary. The current `ProjectStatsResponse` models it as `Dictionary<string, double>`, which is semantically incorrect and prevents clean access to the total bytes value. This plan fixes the model and exposes the total project bytes as a new Prometheus gauge.

## Context

Actual API response structure:
```json
{
  "domain_bytes": {"data": 3.7970113009595e+15},
  ...
}
```

Current model incorrectly treats it as:
```csharp
public Dictionary<string, double> DomainBytes { get; set; } = new();
```

The `domain_bytes.data` value represents the total number of bytes the entire project has processed.

## Steps

### 1. Update `ProjectStatsResponse.cs`

**File:** `src/ArchiveTeam.Exporter.ApiService/Models/ProjectStatsResponse.cs`

- Replace `DomainBytes: Dictionary<string, double>` with a new `DomainBytes` class containing a `Data: double` property.

```csharp
public class ProjectStatsResponse
{
    // ... existing properties ...

    [JsonPropertyName("domain_bytes")]
    public DomainBytesInfo DomainBytes { get; set; } = new();

    // ... rest of properties ...
}

public class DomainBytesInfo
{
    [JsonPropertyName("data")]
    public double Data { get; set; }
}
```

### 2. Update `ProjectService.cs`

**File:** `src/ArchiveTeam.Exporter.ApiService/Services/ProjectService.cs`

#### 2a. Add new Prometheus gauge

Add a new gauge field alongside existing gauges:

```csharp
private readonly Gauge _projectTotalBytesGauge;
```

Initialize it in the constructor:

```csharp
_projectTotalBytesGauge = Metrics
    .CreateGauge(
        "archiveteam_project_total_bytes",
        "Total bytes processed by the project. Data is cached; see archiveteam_cache_last_refresh_timestamp_seconds for cache age.",
        new GaugeConfiguration
        {
            LabelNames = ["name"]
        });
```

#### 2b. Populate the gauge in `GetProjectGaugesAsync`

After setting the other project gauges (`_projectItemsOutGauge`), add:

```csharp
_projectTotalBytesGauge
    .WithLabels(name)
    .Set(stats.DomainBytes.Data);
```

## Files Changed

1. `src/ArchiveTeam.Exporter.ApiService/Models/ProjectStatsResponse.cs` — replace `Dictionary<string, double> DomainBytes` with `DomainBytesInfo DomainBytes`
2. `src/ArchiveTeam.Exporter.ApiService/Services/ProjectService.cs` — add `_projectTotalBytesGauge` and populate it from `stats.DomainBytes.Data`
